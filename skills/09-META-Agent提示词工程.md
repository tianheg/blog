# 元技能09: Agent提示词工程 — 方法论驱动的AI任务设计

**定位**: 指导如何设计高质量的Agent提示词，使AI能够自主完成复杂任务。

## 一、Agent提示词的五层结构

```
L5: 输出规范（JSON schema / Markdown格式）
L4: 验证步骤（自检清单 / lint命令）
L3: 执行方法（SKILL文档 / 错误模式表）
L2: 任务约束（边界条件 / 禁止操作）
L1: 任务目标（要做什么 / 成功标准）
```

## 二、SKILL文档设计原则

| 原则 | 说明 |
|------|------|
| **表格化优于自然语言** | Agent易快速扫描 |
| **实例优于抽象规则** | 可直接模式匹配 |
| **分层组织优于平铺** | 便于快速定位 |
| **版本控制与增量更新** | 追踪演化历史 |

## 三、提示词模板

### 数据提取任务

```markdown
你是XXX提取专家。从文件`{input_file}`中提取{数据类型}。

## 任务目标
提取所有{数据类型}，每个包含{字段列表}。

## 方法论
参考：`skills/SKILL_XX.md`
关键规则：
- {规则1}
- {规则2}

## 输出格式
JSON数组，schema见下：
```json
[
  {
    "field1": "类型+说明",
    "field2": "类型+说明"
  }
]
```

## 质量基线
- 覆盖率：{百分比}
- 准确率：{百分比}

## 验证
- [ ] 所有{数据类型}都已提取
- [ ] 所有必填字段非空
- [ ] 输出JSON格式正确
```

## 四、迭代模式

| 阶段 | 策略 | 质量目标 |
|------|------|----------|
| 冷启动 | 广度优先 | 发现模式，积累错误 |
| 模式积累 | 基于已知模式 | 修正量减半 |
| 收敛验证 | 严格标准 | 新模式=0，修正量<5% |

## 五、常见陷阱

| 陷阱 | 症状 | 解决 |
|------|------|------|
| 提示词过长 | Agent只处理前半部分 | 关键指令前置+分步重复 |
| 质量基线过严 | 修正量远低于预期 | 明确"遗漏也是错误" |
| JSON格式错误 | 输出无法解析 | 提供完整schema+示例 |
| 无限循环 | Agent重复执行 | 明确终止条件 |

## 六、总结

**Agent提示词工程的核心**:
- 五层结构，层层递进
- SKILL文档驱动，表格化、版本化
- 质量基线明确，迭代模式清晰
- 防御性设计，前置关键指令

**核心洞察**: 好的Agent提示词不是"更长"，而是更结构化、更可执行、更可验证。
