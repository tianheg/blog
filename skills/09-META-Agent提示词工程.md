# 元技能09: Agent 提示词工程 — 方法论驱动的 AI 任务设计

## Agent 执行摘要

**一句话**：设计结构化的 Agent 提示词，使 AI 能够自主完成复杂任务并输出可验证的结果。
**何时使用**：需要让 Agent 自主执行批量审查、内容生成、数据提取等任务时。
**核心步骤**：
1. 按五层结构构建提示词：L1 目标 → L2 约束 → L3 方法 → L4 验证 → L5 输出规范
2. 嵌入完整 SKILL 文档（不要摘要）
3. 明确质量基线和终止条件
4. 要求输出 JSON 修正建议 + 新错误模式

## 检查清单

- [ ] 五层结构完整（目标、约束、方法、验证、输出规范）
- [ ] 嵌入了完整的 SKILL 文档或错误模式表
- [ ] 明确了质量基线（预期修正量范围）
- [ ] 要求每处修正附 reasoning 字段
- [ ] 要求提取新错误模式（new_patterns）
- [ ] 提供了完整的输出 JSON Schema + 示例
- [ ] 明确了终止条件（何时停止迭代）

## 详细参考

### 五层结构

```
L1: 任务目标（要做什么 / 成功标准）
L2: 任务约束（边界条件 / 禁止操作）
L3: 执行方法（SKILL 文档 / 错误模式表）
L4: 验证步骤（自检清单 / lint 命令）
L5: 输出规范（JSON schema / Markdown 格式）
```

### 设计原则

| 原则 | 说明 |
|------|------|
| 表格化优于自然语言 | Agent 易快速扫描 |
| 实例优于抽象规则 | 可直接模式匹配 |
| 分层组织优于平铺 | 便于快速定位 |
| 防御性设计 | 关键指令前置，重复重要约束 |

### 提示词模板

#### 模板 1: 内容审查任务

```markdown
你是 {项目} 内容质量审查员。对以下文件执行深度反思。

## L1: 任务目标
审查文件质量，发现并修正错误，使内容符合 SKILL 规范。

## L2: 任务约束
- 不修改原文字符，只修改标注/分类
- 每处修正必须附理由
- 低于质量基线时，二次复查

## L3: 执行方法
参考 SKILL 文档：`skills/SKILL_XX.md`
已知错误模式：
| 错误模式 | 正确形式 | 说明 |
|----------|----------|------|
| ... | ... | ... |

## L4: 验证步骤
- [ ] 所有已知错误模式已检查
- [ ] 新发现的问题已记录
- [ ] 输出 JSON 格式正确

## L5: 输出规范
```json
{
  "corrections": [
    {
      "id": "内容ID",
      "field": "字段名",
      "old_value": "原值",
      "new_value": "新值",
      "reasoning": "修正理由"
    }
  ],
  "new_patterns": ["新发现的错误模式"],
  "stats": {
    "total_checked": 数量,
    "corrections_count": 修正数,
    "confidence": "high/medium/low"
  }
}
```

## 质量基线
- 正常文章预期 X-Y 处修正
- 覆盖率：{百分比}
- 准确率：{百分比}
```

#### 模板 2: 数据提取任务

```markdown
你是 {领域} 提取专家。从文件 `{input_file}` 中提取 {数据类型}。

## L1: 任务目标
提取所有 {数据类型}，每个包含 {字段列表}。

## L2: 任务约束
- 不遗漏任何符合条件的项
- 不提取不符合条件的项
- 字段缺失时标注为 null，不猜测

## L3: 执行方法
参考：`skills/SKILL_XX.md`
关键规则：
- {规则1}
- {规则2}

## L4: 验证步骤
- [ ] 所有 {数据类型} 都已提取
- [ ] 所有必填字段非空
- [ ] 输出 JSON 格式正确

## L5: 输出规范
```json
[
  {
    "field1": "类型+说明",
    "field2": "类型+说明"
  }
]
```

## 质量基线
- 覆盖率：{百分比}
- 准确率：{百分比}
```

#### 模板 3: 代码重构任务

```markdown
你是 {语言} 代码审查专家。重构 `{file_path}` 中的代码。

## L1: 任务目标
提高代码可读性和可维护性，不修改业务逻辑。

## L2: 任务约束
- 保持原有功能不变
- 所有公开接口签名不变
- 不引入新依赖

## L3: 执行方法
重构规则：
| 规则 | 示例 |
|------|------|
| 提取重复代码为函数 | ... |
| 命名规范化 | ... |

## L4: 验证步骤
- [ ] 重构前后运行测试，全部通过
- [ ] 无功能变更

## L5: 输出规范
```json
{
  "changes": [
    {
      "location": "行号/函数名",
      "type": "提取函数/重命名/简化",
      "before": "原代码片段",
      "after": "新代码片段",
      "reasoning": "理由"
    }
  ]

```

### 常见陷阱

| 陷阱 | 症状 | 解决 |
|------|------|------|
| 提示词过长 | Agent 只处理前半部分 | 关键指令前置 + 分步重复 |
| 质量基线过严 | 修正量远低于预期 | 明确"遗漏也是错误" |
| JSON 格式错误 | 输出无法解析 | 提供完整 schema + 示例 |
| 无限循环 | Agent 重复执行 | 明确终止条件 |
