# 元技能06: SKILL 优化与演化 — 方法论文档的持续改进

## Agent 执行摘要

**一句话**：SKILL 本身也是总结出来的，在反思和迭代中不断进化，从粗糙规则成长为精确方法论。
**何时使用**：发现 SKILL 文档有漏洞、需要支持新场景、或从错误中学习改进时。
**核心步骤**：
1. 记录失败案例（20-50 个），统计错误分布
2. 提取共性模式，更新错误模式表
3. 按三层模型更新 SKILL：L1 执行摘要 → L2 核心规则 → L3 详细参考
4. 验证：Agent 使用更新后的 SKILL，准确率是否提升

## 检查清单

- [ ] 失败案例已记录（≥20 个）
- [ ] 共性模式已提取并表格化
- [ ] SKILL 文档按三层结构组织
- [ ] 更新后 Agent 准确率有提升
- [ ] 版本变化有简要记录（非详细历史）

## 详细参考

### SKILL 生命周期

```
冷启动（5-10 条规则）
    ↓ 试点发现漏洞
模式积累（15-30 条规则）
    ↓ 多轮反思沉淀
稳定态（30-50 条规则 + 工具链）
    ↓ 新场景出现
演化升级（规则重构 + 新增模块）
```

### 三层加载模型（本文档自身即示范）

| 层级 | 内容 | 长度 | 用途 |
|------|------|------|------|
| L1 执行摘要 | 一句话 + 何时用 + 核心步骤 | ≤30 行 | Agent 首次读取 |
| L2 核心规则 | 检查清单 + 关键表格 | ≤100 行 | 操作参考 |
| L3 详细参考 | 模板、示例、原理 | 剩余部分 | 按需深入 |

### 质量指标

| 指标 | 定义 | 目标 |
|------|------|------|
| Agent 遵循率 | Agent 按 SKILL 执行的比例 | >90% |
| 首次通过率 | 不触发反思即正确的比例 | >70% |
| 规则覆盖率 | 已知错误中被规则覆盖的比例 | >80% |

### 更新原则

- **增量更新**：每轮反思后追加新模式，不删除旧规则（标记废弃）
- **表格化**：新规则必须附示例，用 Markdown 表格
- **验证闭环**：更新后必须验证，准确率不升则回滚
