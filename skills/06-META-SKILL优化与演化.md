# 元技能06: SKILL优化与演化 — 方法论文档的持续改进

**核心思想**: **"SKILL本身也是总结出来的，在反思中不断积累和抽象"**

## 一、SKILL的生命周期

### 阶段1: 诞生（从日志到SKILL）

```
阶段0: 冷启动 → 手工样本 + 决策日志
    ↓
阶段1: MVP规则 → 5-10条基本规则
    ↓
阶段2: 试点扩展 → 发现问题 → 记录日志
    ↓
阶段3: 模式总结 → 形成SKILL v1.0
```

### 阶段2: 成长（多轮反思与积累）

```
全量应用 + 第一轮反思 → 发现新模式
    ↓
更新SKILL v1.5 → 第二轮反思
    ↓
持续迭代 → SKILL日趋完善
```

### 阶段3: 成熟（抽象与简化）

**问题**: SKILL越来越长（v1.5达5000字）
**解决方案**: 结构化拆分

```
SKILL_03_实体构建.md (总览, 500字)
  ├─ SKILL_03a_实体标注.md (核心规则, 2000字)
  ├─ SKILL_03b_实体消歧.md (专项, 1500字)
  ├─ SKILL_03c_实体标注反思.md (方法论, 2000字)
  └─ SKILL_03d_渲染与发布.md (衔接, 500字)
```

## 二、渐进载入（Progressive Loading）

**原理**: 不要一次性加载所有SKILL，按需载入

**三层载入模型**:

| 层级 | 内容 | 长度 | 场景 |
|------|------|------|------|
| Layer 1 | 总览（必读） | 500字 | 所有任务 |
| Layer 2 | 核心规则（按需） | 2000字 | 简单任务 |
| Layer 3 | 专项深入（按需） | 各1500字 | 复杂任务 |

**收益**: 简单任务节省75% tokens，复杂任务节省60% tokens

## 三、SKILL文档质量指标

| 指标 | 目标 | 检测方法 |
|------|------|----------|
| 规则覆盖率 | >90% | 测试案例覆盖 |
| 规则冲突率 | <5% | 冲突检测脚本 |
| Agent成功率 | >90% | 执行测试 |
| 文档可读性 | <5000字 | 字数统计 |
| 规则数量 | <15条 | 按类型拆分 |

## 四、版本管理与演化

### 版本演化记录

```
v0.1 (2024-10-01): 冷启动MVP
  - 5条基本规则
  - 覆盖率: 70%

v1.0 (2024-11-05): 试点总结
  - 10条规则(+5条)
  - 覆盖率: 85%

v1.5 (2024-12-10): 第一轮反思
  - 15条规则(+5条)
  - 引入新系统
  - 覆盖率: 90%

v2.0 (2025-01-15): 结构化拆分
  - 精简为5条核心规则
  - 详细规则拆分
  - 覆盖率: 92%
```

### 收敛判断标准

- 新模式发现 < 5条/轮
- 修正数量递减
- 准确率稳定（标准差<1%）
- 规则数量稳定（连续3轮不变）

## 五、SKILL设计原则

1. **表格化优于自然语言** — Agent易解析
2. **实例优于抽象规则** — 可直接模式匹配
3. **分层组织优于平铺** — 便于快速定位
4. **版本控制与增量更新** — 追踪演化历史

## 六、跨项目SKILL迁移

### 可迁移性标记

| 规则 | 可迁移性 | 说明 |
|------|----------|------|
| 基本语法 | ★★★★★ | 通用 |
| 消歧策略 | ★★★★☆ | 高度通用，需调整参数 |
| 词表映射 | ★★★☆☆ | 中等，需重建数据 |

### 迁移流程

1. 复制SKILL到新项目
2. 试点应用，记录失败案例
3. 生成差异清单
4. 调整适配
5. 验证通过

## 七、总结

**SKILL优化与演化的核心**:
- 从实践中来，持续抽象
- 结构化拆分，渐进载入
- 版本管理，记录演化
- 质量度量，收敛判断

**核心洞察**: SKILL不是事先设计的完美方案，而是从失败中学习的结果。
