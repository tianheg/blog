# 元技能12: 数据融合 — 多源数据对齐与消歧

**核心思想**: **"来自不同工序的结果，需要对齐、消歧、去重，形成统一的知识视图"**

## 一、融合层次

| 层次 | 内容 | 压缩比 |
|------|------|--------|
| L1 字符串 | 归一化、去重、别名映射 | - |
| L2 实体 | 消歧、规范化、别名合并 | 10x |
| L3 事件 | 跨文档合并、多源整合 | 100x |
| L4 SKU | 结构化知识单元拼接 | 1,000x |
| L5 知识图谱 | 全局一致性、关系网络 | 10,000x |

## 三、实体层面融合

### 四层启发式消歧
1. **前置标记** — 模式: &state&@short_name@
2. **邻近标记** — 在±80字符窗口内查找
3. **文档主题** — 根据文档类型推断
4. **共现全名** — 查找包含短名的全名

## 六、总结

**数据融合的核心**:
- 分层融合: 字符串→实体→事件→SKU→知识图谱
- 多源验证: 跨文档、跨数据源交叉验证
- 冲突记录: 不强制解决，保留所有观点
- 人工覆盖: 关键决策保留人工修正机制

**核心洞察**: 融合的目的不是消除差异，而是结构化地呈现差异，让用户基于完整信息做出判断。
