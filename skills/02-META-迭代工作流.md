# 元技能02: 迭代工作流 — 从冷启动到收敛的执行策略

## Agent 执行摘要

**一句话**：分阶段推进执行，先快速覆盖全部再精修，通过多轮迭代使质量收敛。
**何时使用**：任务复杂、规则模糊、需要质量逐步提升时。
**核心步骤**：
1. 判断当前所处阶段（见下方决策树）
2. 按阶段策略执行：P0 粗粒度 → P1 中粒度 → P2 收敛 → P3 终态
3. 每轮产出错误模式表（核心资产），而非仅修正数据
4. 达到收敛标准时停止

**当前阶段快速判断**：
- 无 prior 错误模式，SKILL 文档几乎为空 → **P0 冷启动**
- 有已知模式，上次修正率 > 30% → **P1 模式积累**
- 上次修正率 < 5%，lint 通过，无新模式 → **P3 收敛**

## 检查清单

- [ ] 采用广度优先（先覆盖全部再精修），非深度优先
- [ ] 每轮产出错误模式表（≥5 条新发现）
- [ ] 修正量呈指数衰减趋势
- [ ] 质量目标逐轮收紧（P0 宽松 → P3 严格）
- [ ] 超过 5 轮不收敛则人工介入

## 详细参考

### 四阶段模型

| 阶段 | 名称 | 准确率目标 | 修正量预期 | 新模式预期 | 核心产出 |
|------|------|-----------|-----------|-----------|----------|
| P0 | 冷启动 | 50-70% | N/A | 20-30 条 | 初版数据 + 错误模式表 |
| P1 | 模式积累 | 80-85% | P0 的 30-50% | <10 条 | 修正数据 + 新模式 |
| P2 | 收敛验证 | 90-92% | P1 的 20-30% | 0 条 | 高质量数据 + 收敛报告 |
| P3 | 终态验证 | 95-97% | <5% | 0 条 | 发布级数据 + 质量报告 |

### 何时进入下一轮

- 修正量仍 > 5%
- 发现新错误模式
- 抽样准确率 < 95%

### 何时停止（必须全部满足）

- [ ] 修正量 < 总量的 5%
- [ ] 新模式数 = 0（连续两轮）
- [ ] Lint 100% 通过
- [ ] 人工抽样准确率 > 95%

### 广度优先 vs 深度优先

**❌ 深度优先（错误）**：
```
文章1 → 完美处理（100%）→ 花费 2 天
文章2 → 完美处理 → 花费 2 天
...到文章 30 时发现：前 29 篇分类标准错了，全部返工
```

**✅ 广度优先（正确）**：
```
Day 1-2: 快速处理全部（60% 准确率）
Day 3: 抽样检查，发现分类标准问题
Day 4: 修正标准，更新 SKILL 文档
Day 5-7: 第二轮处理（基于正确标准）
```

### 错误模式驱动

每轮迭代的核心产出不是"修正数据"，而是"错误模式表"。

```
## 第一轮产出（核心资产）
错误模式表（25 条）:
| ID | 模式 | 频率 | 修正方法 |
|----|------|------|---------|
| P01 | 标题格式不统一 | 12 篇 | 统一 H2 格式 |
| P02 | 代码块缺语言标识 | 全书 | 强制添加 |

修正数据（1010 处）: 次要产出
```

### 过度迭代应对

| 迹象 | 原因 | 解决 |
|------|------|------|
| 超过 5 轮不收敛 | 任务定义或 SKILL 文档有问题 | 人工介入，重新定义 |
| 修正量 20-30% 震荡 | 规则冲突 | 检查 SKILL 文档规则一致性 |
| 每轮 5+ 新模式 | 任务不清晰 | 任务拆分，降低复杂度 |
