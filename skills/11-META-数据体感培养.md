# 元技能11: 数据体感培养 — 直接观察数据的十层体系

**核心思想**: **"任何统计都替代不了对数据的直接观察"**

## 一、十层数据观察体系

| 层级 | 工具/方法 | 观察对象 | 产出 |
|------|----------|----------|------|
| L1 | 差异检测 | 标注前后差异 | 字符级差异报告 |
| L2 | 统计分析 | 词频/n-gram分布 | 高频词表 |
| L3 | 覆盖率分析 | 标注覆盖率/类型分布 | 类型统计表 |
| L4 | 专项扫描 | 特定类型完整性 | 问题清单 |
| L5 | 格式验证 | 数据格式/完整性 | 验证报告 |
| L6 | 逻辑验证 | 跨数据源一致性 | 偏差报告 |
| L7 | 采样检查 | 消歧质量 | 审查报告 |
| L8 | 可视化 | 实体索引/关系图 | 交互式界面 |
| L9 | 工作日志 | 每日修正记录 | Git提交汇总 |
| L10 | 反思总结 | 多轮迭代收敛 | 模式发现文档 |

## 二、核心原则

### 即时可见
- 每个工序的中间结果必须可直接阅读
- Markdown/JSON/TSV格式

### 多维观察
- 统计汇总 + 随机抽样 + 异常检测 + 可视化

### 人工介入
- 定期人工检查，不完全依赖自动化指标

## 三、采样策略

| 策略 | 方法 |
|------|------|
| **分层随机抽样** | 按类型分层，按比例抽样 |
| **边界情况优先** | 主动查看极端值 |
| **错误驱动采样** | 优先检查自动化质检发现的问题 |

## 四、数据观察清单

### 每个工序必做
- [ ] 样本检查（10-20个）
- [ ] 统计汇总（count/avg/min/max）
- [ ] 异常检测（超出2σ的离群值）
- [ ] 随机抽样（50个人工审查）
- [ ] 边界测试（最长/最短/空值）

## 五、总结

**数据体感培养三原则**:
1. **不信任统计，信任样本** — 精度95%不够，要看具体错在哪5%
2. **不信任自动化，信任人工验证** — 自动化找候选，人工最终判断
3. **不信任一次观察，信任多轮迭代** — 每周review，持续观察

**核心洞察**: 数据体感是所有元技能的基础 — 只有深入理解数据，才能设计好的规则、prompt和质检工具。
