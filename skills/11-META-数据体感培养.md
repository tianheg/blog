# 元技能11: 数据体感培养 — 直接观察数据的十层体系

## Agent 执行摘要

**一句话**：任何统计都替代不了对数据的直接观察。培养对数据分布、异常、模式的直觉，才能设计出有效的规则和质检工具。
**何时使用**：需要为大量内容设计分类规则、质检标准、或发现系统性问题时。
**核心步骤**：
1. 从 L1 到 L10 逐层观察数据（至少覆盖 L1-L5）
2. 每层记录 3-5 个关键发现
3. 将发现转化为规则或错误模式，写入 SKILL 文档
4. 每周 review，持续观察

## 检查清单

- [ ] 已直接观察原始数据（非仅看统计报告）
- [ ] 记录了异常值和边界情况
- [ ] 发现了至少 3 个可归纳的模式
- [ ] 观察结论已转化为规则或检查项
- [ ] 定期复查（每周/每轮迭代）

## 详细参考

### 十层观察体系

| 层级 | 观察内容 | Agent 行动 | 本项目示例 |
|------|----------|-----------|-----------|
| L1 存在性 | 文件是否存在、是否为空 | `find content/til -name "*.org" -empty` | 空 TIL 文件 |
| L2 分布 | 分类/标签/长度的分布 | `ls content/til/*/*.org | awk -F/ '{print $3}' | sort | uniq -c` | software/ 占比 60% |
| L3 边界 | 最大值、最小值、异常值 | 扫描最长/最短文件 | 某 TIL 仅 2 行 |
| L4 结构 | 格式是否符合规范 | `./scripts/til-lint.sh` | 缺 `#+END_SRC` |
| L5 模式 | 重复出现的结构或错误 | `grep -r "【重点】" content/til | wc -l` | 50% 文件无重点标记 |
| L6 逻辑 | 跨文件一致性 | 对比同类 TIL 的结构差异 | git-xxx 格式不统一 |
| L7 语义 | 内容质量、歧义 | 采样 10 篇人工阅读 | "重述理解"过于笼统 |
| L8 关联 | 文件间的引用关系 | `grep -o "\[\[.*\]\]" content/til/**/*.org` | 几乎无内部链接 |
| L9 演化 | 时间趋势 | `git log --stat content/til/` | 近期新增 health/ 内容 |
| L10 综合 | 整体健康度评估 | 综合以上指标打分 | 80% 合规，20% 需改进 |

### 关键原则

1. **不看统计就下结论** — 统计会掩盖分布细节
2. **从异常值中学习** — 边界情况最能暴露规则漏洞
3. **不信任一次观察** — 每周 review，持续观察

### 快速观察命令

```bash
# L2 分布：各分类文件夹文件数
for d in content/til/*/; do echo "$(ls "$d"*.org 2>/dev/null | wc -l) $(basename "$d")"; done | sort -rn

# L3 边界：最短和最长的 TIL
find content/til -name "*.org" -exec wc -l {} + | sort -n | head -5
cfind content/til -name "*.org" -exec wc -l {} + | sort -n | tail -5

# L4 结构：缺少 💡 或 📋 的文件
grep -L "💡 重述理解" content/til/*/*.org 2>/dev/null
grep -L "📋 内容主体" content/til/*/*.org 2>/dev/null
```
