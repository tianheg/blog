# 元技能03: 冷启动 — 从零开始构建 AI 驱动工序的方法论

## Agent 执行摘要

**一句话**：在没有先验知识、没有训练数据、没有成熟规则时，通过"少量样本建立直觉 → 提炼规则 → 试点验证 → 全量执行"快速启动。
**何时使用**：面对全新任务类型，无法直接套用现有 SKILL 时。
**核心步骤**：
1. **任务定义**：明确输入/输出/成功标准（写 1 段话 + 1 个 JSON Schema）
2. **手动处理**：处理 3-10 个样本，建立直觉，提炼 5-10 条初步规则
3. **SKILL 文档**：将规则写成最小可用 SKILL（5-10 条规则 + 2-3 个示例）
4. **小规模试点**：Agent 处理 10-20 个样本，对比人工结果，重叠度 >70% 才进入全量
5. **全量执行**：批量处理，记录错误模式

**冷启动完成标志**：
- [ ] 全部数据已处理（不要求完美）
- [ ] 抽样准确率 > 60%
- [ ] 错误模式表已建立（15-30 条）
- [ ] 进入迭代阶段的准备就绪

## 检查清单

- [ ] 任务目标、输入输出、成功标准已明确
- [ ] 手动处理了 3-10 个样本（覆盖简单/中等/复杂）
- [ ] SKILL 文档包含 5-10 条规则 + 2-3 个完整示例
- [ ] 执行了 10-20 个样本的试点，与人工重叠度 >70%
- [ ] 不追求准确率 >90%（这是迭代的目标）

## 详细参考

### 任务定义模板

```json
{
  "task": "任务名称",
  "input": "输入描述",
  "output": "输出描述",
  "format": { "字段": "类型+说明" },
  "success_criteria": "定量/定性指标"
}
```

**示例**（博客文章生成）：
| 问题 | 答案 |
|------|------|
| 要做什么? | 输入关键词/大纲，输出 1500-3000 字完整文章 |
| 为什么做? | 提高内容产出效率，保持风格一致性 |
| 怎样算成功? | 准确率 >80%，人工审核通过 |

### 手动处理四步法

1. **盲标** — 不参考任何规则，凭直觉处理
2. **自我审查** — 检查粒度一致、类型清晰、无遗漏
3. **提炼规则** — 从处理过程中提炼决策规则
4. **交叉验证** — 用第 2-3 个样本验证规则

### SKILL 文档最小可行版本（MVP）

必备内容：
- ✅ 任务目标（1 段）
- ✅ 输出格式（JSON Schema 或模板）
- ✅ 核心规则（5-10 条，每条附 1 个示例）
- ✅ 边界情况（3-5 个困难 case）

规则表达三原则：
1. **具体胜于抽象** — 每条规则可直接应用
2. **示例驱动** — 每条规则至少附 1 个示例
3. **允许模糊** — 冷启动阶段标注"待定"，迭代时决定

### 试点分析维度

1. **与人工处理的对比** — 重叠度、遗漏、多余
2. **输出格式检查** — 是否符合 Schema
3. **粒度一致性检查** — 不同样本处理质量是否一致

**试点后决策**：
- >30% 误解规则 → 重写 SKILL 文档
- 与人工重叠度 <70% → 不进入全量，返回 Phase 2

### 常见陷阱

| 陷阱 | 症状 | 解决 |
|------|------|------|
| 过早优化 | Phase 1 追求完美，每个样本 4-8 小时 | 快速完成（2 小时/样本），迭代时细化 |
| 样本偏差 | 只选简单样本，复杂场景准确率极低 | 刻意包含简单/中等/复杂样本 |
| 跳过试点 | SKILL 后直接全量，发现 20% 格式错误 | 严格执行试点（10-20 个），不通过不进入全量 |
