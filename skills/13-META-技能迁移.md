# 元技能13: 技能迁移 — SKILL跨项目复用方法论

**核心思想**: **"SKILL文档是可迁移的知识资产"**

## 一、迁移层次

| 层次 | 适用场景 | 操作 |
|------|----------|------|
| **Level 1 直接复用** | 目标数据与源数据高度相似 | 复制SKILL，80%直接可用 |
| **Level 2 适配调整** | 有部分差异 | 调整参数/规则，20%需调整 |
| **Level 3 重新设计** | 结构根本不同 | 重新设计部分SKILL |

## 二、迁移流程

### 试点驱动的迁移

```
阶段1: 选择试点（3-5篇）
  ├─ 覆盖多样性（不同类型/难度）
  └─ 简单→中等→复杂
  
阶段2: 直接应用现有SKILL
  ├─ 复制SKILL到新项目
  ├─ 对试点进行标注
  └─ 运行Lint工具，收集错误
  
阶段3: 记录失败案例
  ├─ 创建迁移错误日志
  ├─ 记录20-50个问题
  └─ 统计错误分布
  
阶段4: Agent反思
  ├─ 根因分析
  ├─ 共性模式提取
  └─ 生成修正建议
  
阶段5: 人工审查反思结果
阶段6: 修正SKILL文档
阶段7: 增量验证
阶段8: 扩展部署
```

## 三、迁移成本估算

| 场景 | 相似度 | 预估工作量 | 节省时间 |
|------|--------|-----------|----------|
| 史记→汉书 | 85% | 230小时 | 53% |
| 史记→资治通鉴 | 60% | 370小时 | 50% |
| 第N次迁移 | - | 递减 | 80%+ |

## 四、反思驱动的适配

### 迁移错误日志模板

```markdown
# 源项目→目标项目_迁移日志

## 试点章节: XXX
### 错误1: 类型问题
- **现象**: ...
- **原因**: ...
- **影响范围**: ...
- **修正方案**: ...
- **状态**: 待修正
```

### 反思输出

```markdown
## 根因分析
- 历史时代差异
- 文体结构差异

## 共性模式
- 词表不完整 (40%)
- 文体假设失效 (25%)
- 标注粒度不足 (20%)

## 修正建议（优先级排序）
- P0: 扩展词表
- P1: 参数化文体类型
- P2: 标注设计
```

## 七、总结

**技能迁移的核心**:
- 分层迁移: 直接复用 → 适配调整 → 重新设计
- 试点驱动: 小规模试点发现问题
- 反思循环: 错误日志 → Agent反思 → SKILL修正 → 验证
- 增量验证: 每修正一个问题，立即验证
- Human-in-the-loop: 关键决策需要人工审查

**核心洞察**: SKILL文档不是一次性设计，而是在多项目迁移中持续演化的知识资产。迁移过程本身就是知识提炼的过程。
