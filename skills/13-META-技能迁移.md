# 元技能13: 技能迁移 — SKILL 跨项目复用方法论

## Agent 执行摘要

**一句话**：将已验证的 Skill 从源项目迁移到目标项目，通过试点验证、差异分析、适配调整，实现知识资产复用。
**何时使用**：新项目需要类似能力、或从旧项目提取通用方法论时。
**核心步骤**：
1. **评估相似度**：目标项目与源项目的结构/场景相似度
2. **试点验证**：选 3-5 个样本直接应用现有 Skill，记录失败率
3. **差异分析**：统计失败案例，提取共性模式
4. **适配调整**：修改规则、补充边界情况、更新示例
5. **增量验证**：逐步扩大范围，直到覆盖率达标

## 检查清单

- [ ] 源项目 Skill 已稳定（准确率 >80%）
- [ ] 已评估目标项目相似度（直接复用 / 适配调整 / 重新设计）
- [ ] 试点样本覆盖多样性（简单/中等/复杂）
- [ ] 失败案例已记录并分析根因
- [ ] 适配后的 Skill 经过验证，准确率达标

## 详细参考

### 迁移层次

| 层次 | 适用场景 | 操作 | 工作量 |
|------|----------|------|--------|
| Level 1 直接复用 | 目标与源高度相似 | 复制 SKILL，80% 直接可用 | 10% |
| Level 2 适配调整 | 有部分差异 | 调整参数/规则/示例 | 30% |
| Level 3 重新设计 | 结构根本不同 | 重新设计部分 SKILL | 70% |

### 试点驱动迁移流程

```
阶段1: 选择试点（3-5 个样本）
  └─ 覆盖多样性（不同类型/难度）

阶段2: 直接应用现有 SKILL
  └─ 运行审查，收集错误

阶段3: 记录失败案例
  └─ 创建迁移日志（20-50 个问题）
  └─ 统计错误分布

阶段4: 差异分析
  └─ 根因分类（规则不适用 / 新场景 / 格式差异）
  └─ 提取共性模式

阶段5: 适配 SKILL
  └─ 修改规则、补充示例、调整输出格式

阶段6: 增量验证
  └─ 10 → 50 → 200 → 全量
  └─ 每步统计准确率
```

### 迁移日志模板

```markdown
## SKILL 迁移日志

**源项目**: {project_a}
**目标项目**: {project_b}
**迁移日期**: YYYY-MM-DD

### 试点结果
- 样本数: 5
- 直接通过率: 3/5 (60%)
- 主要失败类型: 格式差异 (2)

### 适配修改
| 规则 | 原表述 | 修改后 | 原因 |
|------|--------|--------|------|
| 文件名 | `{prefix}-{desc}.org` | `{topic}-{desc}.org` | 目标项目无前缀体系 |

### 验证结果
- 适配后试点通过率: 5/5 (100%)
- 扩展验证 (50 样本): 48/50 (96%)
```

### Blog 场景示例

**场景**：将本项目的 `SKILL_TIL_编写规范.md` 迁移到一个新的笔记项目

**评估**：Level 2 适配调整
- 相同：都是知识笔记、都需要结构化、都用 Markdown/Org
- 不同：新项目不用 Org Mode，用 Markdown；无 software/ 前缀体系

**适配**：
1. 将 Org 模板转为 Markdown 模板
2. 移除 software/ 前缀要求，改用标签分类
3. 保留 "重述理解 + 内容主体" 的核心结构
4. 更新示例为 Markdown 格式

---

**核心洞察**：SKILL 文档不是一次性设计，而是在多项目迁移中持续演化的知识资产。迁移过程本身就是知识提炼的过程。
