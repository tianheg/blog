# 元技能01: 反思 — Agent 驱动的质量审查通用方法论

## Agent 执行摘要

**一句话**：对 AI 生成内容进行系统性质量审查与迭代修正，通过多轮迭代使质量收敛。
**何时使用**：生成内容后、发布前，或发现系统性错误需要批量修正时。
**核心步骤**：
1. **预备**：明确质量标准，编写/更新 SKILL 文档中的错误模式表
2. **侦察**：抽样审查 10-20 篇，统计错误分布，确定审查策略
3. **执行**：Agent 批量审查，输出 JSON 修正建议，应用修正
4. **验证**：运行 lint，对比修正前后统计，判断是否收敛

**收敛标准**（必须全部满足）：
- [ ] 修正量 < 总量的 5%
- [ ] 新模式数 = 0（连续两轮）
- [ ] Lint 100% 通过

## 检查清单（P0 必查）

- [ ] 质量标准已量化（明确"好"的标准）
- [ ] 已知错误模式已表格化（错误 → 正确 → 说明）
- [ ] Agent Prompt 中嵌入了完整 SKILL 文档
- [ ] 每处修正附 reasoning 字段
- [ ] 修正后运行 lint 验证
- [ ] 抽样复查 5-10%

## 详细参考

### 审查粒度选择

| 粒度 | 适用场景 | 选择时机 |
|------|----------|----------|
| 按篇 | 单篇语境误判、文章特有错误 | 第一轮打底 |
| 按类型 | 同一类错误在全书中重复 | 发现系统性模式后 |
| 全局 | 跨章交叉验证、一致性审查 | 最后收尾 |

### 四阶段工作流

**Phase 0: 预备**
- 产出：SKILL 文档（错误模式表 + 审查清单）+ lint 工具
- 错误模式表格式：| 错误模式 | 正确形式 | 说明 |

**Phase 1: 侦察**
- 抽样 10-20 篇，覆盖不同体裁
- 统计错误分布（按篇/按类型）
- 若某类错误在 >50% 文章中重复 → 采用按类型反思

**Phase 2: 执行**

Agent 提示词模板：

```markdown
你是内容质量审查员。对以下文件执行深度反思。

## 输入
- 文件路径: {file_path}
- SKILL 文档: skills/SKILL_XX.md
- 质量基线: 预期修正量 X-Y 处

## 任务
1. 读取 SKILL 文档，了解所有已知错误模式
2. 逐段审查：
   - 格式问题（lint 结果）
   - 已知错误模式（SKILL 表格）
   - 新发现的问题
3. 输出 JSON:
   {
     "corrections": [
       {
         "id": "内容ID",
         "field": "字段名",
         "old_value": "原值",
         "new_value": "新值",
         "reasoning": "修正理由"
       }
     ],
     "new_patterns": ["新发现的错误模式描述"],
     "stats": {
       "total_checked": 数量,
       "corrections_count": 修正数,
       "confidence": "high/medium/low"
     }
   }

## 约束
- 不修改原文字符，只修改标注/分类
- 每处修正必须附理由
- 低于质量基线时，二次复查
```

**Phase 3: 验证**
1. 格式验证（必须通过）
2. 统计对比（修正前后质量指标提升）
3. 抽样复查（5-10%）
4. 收敛判断（修正量 < 5%、新模式 = 0、lint 100% 通过）

### JSON 修正格式

```json
{
  "article": "文件名",
  "round": 1,
  "corrections": [
    {
      "id": "唯一ID",
      "type": "错误类型",
      "field": "字段名",
      "old_value": "修正前",
      "new_value": "修正后",
      "reasoning": "理由（必填）",
      "confidence": "high/medium/low"
    }
  ],
  "new_patterns": [
    {
      "pattern": "错误模式描述",
      "frequency": "估计频率",
      "solution": "修正方法"
    }
  ],
  "stats": {
    "total_items": 123,
    "corrections_count": 45,
    "correction_rate": 0.366
  }
}
```

### 迭代控制

| 轮次 | 策略 | 质量目标 |
|------|------|----------|
| 第一轮 | 广度优先 | 全覆盖，宁可多修也不漏修 |
| 第二轮 | 精度提升 | 修正量预期减半 |
| 第三轮 | 收敛验证 | 修正量 < 20%（相对第二轮） |
| 第 N 轮 | 终止条件 | 修正量 < 5% 或新模式数 = 0 |

**过度迭代迹象**（需人工介入）：
- 超过 5 轮仍不收敛 → 重新审视任务定义
- 修正量在 20-30% 之间震荡 → 检查 SKILL 文档规则冲突
- 每轮都发现 5+ 条新模式 → 任务拆分，降低复杂度

### 错误模式积累原则

- **表格化沉淀** — Markdown 表格，非自然语言
- **三列结构** — 错误模式 | 正确形式 | 说明/理由
- **实例优先** — 每种模式附至少 1 个实际案例
- **版本控制** — 每轮反思后更新 SKILL 文档
