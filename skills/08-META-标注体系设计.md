# 元技能08: 标注体系设计 — 语义标记符号的工程化设计原则

**定位**: 指导如何设计语义标注体系（Token标记、类型分类、消歧规则）。

## 一、标记符号的四种范式

| 范式 | 示例 | 优势 | 适用场景 |
|------|------|------|----------|
| XML/SGML | `<person>名称</person>` | 标准化、可嵌套 | 学术数字人文 |
| HTML/CSS | `<span class="person">` | 渲染直接 | Web发布 |
| Markdown扩展 | `[名称]{.person}` | 轻量、兼容 | Markdown增强 |
| **自定义Token** | `〖@名称〗` | 极简、Git友好 | 大规模标注 |

## 二、本项目的选择：〖TYPE content〗

### 设计决策
- **白色六角括号〖〗** — CJK符号，视觉突出，极少冲突
- **单字符类型标记** — @=;#%&'^~•!+$?{:[_ 等18种
- **内联消歧语法** — | 分隔显示名与规范名

### 对比示例

```
XML: <person>黄帝</person>者，<person>少典</person>之子
Token: 〖@黄帝〗者，〖@少典〗之子

字符膨胀: XML +300%，Token +30%
```

## 三、类型分类原则

### 三个维度
1. **封闭性** — 可枚举 vs 不可枚举
2. **粒度** — 粗粒度 vs 细粒度
3. **正交性** — 类型间无歧义

### 示例分类

| 符号 | 类型 | 示例 |
|------|------|------|
| @ | 人名 | 〖@张三〗 |
| = | 地名 | 〖=北京〗 |
| ; | 职位 | 〖;工程师〗 |
| # | 身份 | 〖#CEO〗 |
| % | 时间 | 〖%2024年〗 |
| & | 组织 | 〖&公司名〗 |
| $ | 数量 | 〖$100万〗 |

## 四、消歧规则设计

### 三层消歧
1. **内联消歧** — 〖@显示名|规范名〗
2. **映射表消歧** — 章节级别的映射JSON
3. **全局推理消歧** — 跨章节分析

### 内联消歧语法

```markdown
原文: 台弑君而自立
标注: 〖@台|吕台〗弑君而自立
渲染: <span title="吕台">台</span>弑君而自立
```

## 五、演化策略

### 版本迁移
```
v1.0: @名称@ → v2.0: 〖@名称〗
v2.0: 4类 → v2.5: 8类 → v3.0: 18类
```

### 向后兼容
- 新版本渲染旧版本标注
- 提供迁移脚本

## 六、总结

**标注体系设计的核心**:
- 符号轻量、不冲突、Git友好
- 类型正交、粒度适中、可演化
- 消歧分层、边界明确、向后兼容

**核心洞察**: 标注体系不是一次性设计，而是随认知深化而迭代演化。
