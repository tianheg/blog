## 调用策略

1. **项目内容操作**（写 TIL、改文章）→ 先读 `SKILL_TIL_编写规范.md`
2. **内容质量审查** → 读 `01-META-反思.md`
3. **任务规划/分解** → 读 `03-META-冷启动.md` + `04-META-柳叶刀方法.md`
4. **改进 Skill 本身** → 读 `06-META-SKILL优化与演化.md`
5. **其他元技能** → 按需读取，非必须

> **Token 优化建议**：元技能文件较长（200-400 行），优先读取"核心洞察"和"检查清单"部分，略过背景论述。
