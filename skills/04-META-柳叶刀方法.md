# 元技能04: 柳叶刀方法 — 精细分解与混合解决方案

**定位**: 将复杂AI任务进行精细分解、并针对每个子问题选择最优解决方案（大模型/规则/代码）的方法论。

## 一、核心思想

> **"手术刀式精准切分，而非大锤式端到端"**

- **模块化分解**: 将复杂流程拆解为可独立优化的阶段
- **细粒度建模**: 多个专门化的微模型协同工作
- **混合驱动**: LLM的泛化能力 + 规则的精确性 + 代码的效率
- **快速迭代**: 以小时为单位的增量改进

## 二、任务分工决策树

```
问题分析
    ├─ 是否有明确规则？
    │   ├─ 是 → 【规则+代码】（如格式校验、类型过滤）
    │   └─ 否 ↓
    ├─ 是否结构化匹配？
    │   ├─ 是 → 【代码算法】（如实体共现、时间对齐）
    │   └─ 否 ↓
    ├─ 是否需要语义理解？
    │   ├─ 是 → 【LLM】（如因果推理、情感分析）
    │   └─ 否 ↓
    ├─ 是否可安全自动化？
    │   ├─ 是 → 【规则+LLM联合】（规则筛选 + LLM确认）
    │   └─ 否 → 【人工审查】（保守策略）
```

## 三、混合解决方案矩阵

| 任务特征 | 推荐方案 | 示例 |
|----------|----------|------|
| 有标准答案的查表任务 | 代码 | 日期查询、常量映射 |
| 可枚举规则的模式匹配 | 规则+Regex | 格式提取、关键词匹配 |
| 需要上下文推理 | LLM | "三年后"的绝对时间 |
| 结构化图计算 | 代码 | 关联分析、网络分析 |
| 多源数据一致性检查 | 规则+代码 | 数据验证、约束检查 |
| 歧义消解（有先验） | 规则→LLM→人工 | 命名实体消歧 |
| 语义分类（无标准） | LLM | 内容分类、情感分析 |
| 批量重复任务 | 自动化优先 | 批量处理流水线 |
| 高风险决策 | 人工确认 | 关键业务决策 |

## 四、LLM与规则的协同模式

### 模式1: 规则前置筛选 + LLM精炼

```
全部数据（100万条）
  ↓ [规则筛选]: 共享特征/时间接近
候选数据（5000条） ← 减少99.5%
  ↓ [LLM推理]: 语义判断
最终结果（500条）
```

**效益**: 节省API成本、提高精度、可解释

### 模式2: LLM生成候选 + 规则验证

```
LLM生成候选
  ↓ [启发式规则验证]
    - 格式检查
    - 值域约束
    - 频率过滤
  ↓ [自动确认]
安全结果（80%）
  ↓ [人工审查]
存疑结果（20%）
```

### 模式3: 规则兜底 + LLM补全

```
def process_data(data):
    # 第一层: 规则查表（40%覆盖）
    if data in RULE_TABLE:
        return RULE_TABLE[data]

    # 第三层: LLM语义推理（+25%覆盖）

```

## 五、MECE分解原则

**Mutually Exclusive, Collectively Exhaustive**（互斥且完备）

```
任务X
  ├─ 子任务A（规则可解决）
  │   └─ 方案: 规则+代码
  ├─ 子任务B（需语义理解）
  │   └─ 方案: LLM
  └─ 子任务C（高风险）
      └─ 方案: 人工
```

**检查清单**:
- [ ] 子任务之间无重叠（避免重复处理）
- [ ] 子任务覆盖全部场景（无遗漏）
- [ ] 每个模块有明确输入/输出/验证标准

## 六、成本效益分析

**示例**: 跨文档实体关联（867条关系）

| 方案 | 成本 | 时间 | 精度 | 召回 |
|------|------|------|------|------|
| 纯LLM | $41,000 | 11小时 | 87% | 92% |
| 纯规则 | $50 | 5分钟 | 95% | 68% |
| **混合** | **$380** | **1小时** | **94%** | **89%** |

**结论**: 混合方案在精度/召回/成本三维度达到最优平衡。

## 七、反模式（避免）

| 反模式 | 表现 | 后果 | 解决 |
|--------|------|------|------|
| 过度依赖LLM | 所有任务都用LLM | 成本爆炸、速度慢、不稳定 | 能用规则就用规则 |
| 规则过度工程 | 为每个边界写规则 | 维护困难、泛化能力差 | 分层策略 |
| 黑箱流水线 | 各模块独立，无中间输出 | 调试困难、误差传播 | 每阶段保存中间文件 |
| 盲目端到端 | 单一大模型解决所有问题 | 数据需求大、可解释性差 | 模块化流水线 |

## 八、总结

**柳叶刀方法的核心**:
1. **精细分解** — 复杂任务→可独立优化的模块
2. **混合驱动** — LLM/规则/代码各取所长
3. **MECE原则** — 互斥完备，无遗漏无重叠
4. **成本意识** — 不盲目用LLM，优先规则+代码
5. **可追溯性** — 标记来源，支持审计调试
6. **迭代优化** — 从规则冷启动→LLM补全→规则固化

**核心洞察**: 最优方案不是设计出来的，是从对比实验中总结出来的。
