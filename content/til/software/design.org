*** Ten Principles of Good Design (Dieter Rams)
:PROPERTIES:
:CUSTOM_ID: dieter-rams
:END:
https://sharing.ia.net/presenter/4facc352d646499b9b8d3bb6c2fbad2b/view

Dieter Rams 认为好的设计应该是：

1. **Innovative** - 创新。技术发展总是为创新设计提供新机会。
2. **Useful** - 有用。产品不仅要满足功能标准，还要满足心理和美学标准。
3. **Aesthetic** - 美观。产品的美学质量与其有用性密不可分。
4. **Understandable** - 易懂。好的设计能让产品自己说话，最好的设计是不言自明的。
5. **Unobtrusive** - 不显眼。产品是工具，不是装饰品或艺术品。
6. **Honest** - 诚实。不夸大产品的创新、功能或价值。
7. **Long-lasting** - 持久。避免时尚，因此不会显得过时。
8. **Thorough down to the last detail** - 极致细节。没有什么是随意或偶然的。
9. **Ecological** - 环保。设计为环境保护做贡献。
10. **Minimal** - 少即是好。专注于本质，回归纯净与简单。

*** Inclusive Design Principles
:PROPERTIES:
:CUSTOM_ID: inclusive-design
:END:
https://inclusivedesignprinciples.org/

包容性设计原则「以人为先」。满足人的短暂需要、长期需要、特定情形的需要，包括残疾人乃至其他身体心智健全的人。

原则：
1. 提供合适的体验
2. 考虑具体情况
3. 保持一致
4. 提供控制方法
5. 提供选择
6. 对不同优先级的内容进行区分
7. 添加值

** RESTful API Design
:PROPERTIES:
:CUSTOM_ID: restful-api
:END:
设计 RESTful API 的基本原则和最佳实践：

1. **Use HTTP Methods Appropriately** - 使用标准 HTTP 方法（GET, POST, PUT, DELETE）
2. **Resource-Oriented** - 围绕资源组织 API，使用名词而不是动词
3. **Use URIs to Identify Resources** - 使用层级 URI 表示资源关系，如 `/users/{userId}/orders`
4. **HTTP Status Codes** - 使用合适的 HTTP 状态码（200 OK, 201 Created, 404 Not Found, 400 Bad Request）
5. **Consistent and Predictable** - 保持 API 设计的一致性和可预测性
6. **Stateless** - 每个请求包含所有必要信息，不维护客户端状态
7. **HATEOAS** - 在响应中提供超媒体链接，引导客户端进行下一步操作
8. **Support Content Negotiation** - 使用 `Accept` 头支持内容协商
9. **Versioning** - 对 API 进行版本管理
10. **Security** - 实现适当的认证和授权机制
