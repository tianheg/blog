** 📋 内容主体

文章将40个概念分为6大核心领域，以下是各领域的核心要点：

*** 1. 算法与数据结构

| 概念 | 类比解释 | 核心要点 |
|------|----------|----------|
| Big O 复杂度 | 网购蓝光碟 vs 下载电影 | O(1)输入不影响时间，O(n)时间与输入成正比，衡量算法可扩展性 |
| 排序算法 | - | 通过可视化动画理解不同排序策略 |
| 递归 | 电影院问排数 | 函数调用自身，直到达到基准条件后逐层返回 |
| 大数据 | 水管漏水：水桶→水管工→大坝 | 数据量/速度超出传统工具处理能力，需要全新方法 |
| 数据结构 | - | 【重点】数组、树、栈、队列、图、哈希表、链表、堆 |

*** 2. 人工智能

| 概念 | 类比解释 | 核心要点 |
|------|----------|----------|
| 贪心算法 | 爬山只选最陡的路 | 只选当前最优，不考虑全局，可能错过更好的解 |
| 爬山算法 | 雾中爬山，记录最高海拔 | 基于当前最优解生成邻近解，可能陷入局部最优 |
| 模拟退火 | 登珠峰时偶尔下坡探索 | 早期允许较差移动，随"温度"降低逐渐收敛 |
| 动态规划 | 1+1+1+... 记住之前的结果 | 【重点】记忆化（memoization）存储子问题结果，避免重复计算 |
| P vs NP | 乘法容易，因式分解难 | P问题易解，NP问题易验证难解。若P=NP将改变世界（密码破解、蛋白质折叠等） |

*** 3. 计算机架构

- 计算机工作原理：通过层层抽象将0/1转换为程序，就像开车不需要懂引擎原理
- 停机问题：无法判断程序是否会停止运行（不可判定问题）

*** 4. 并发

| 概念 | 类比解释 | 核心要点 |
|------|----------|----------|
| 并发 | 秘书同时处理电话、会议、文档 | 任务在重叠时间段内运行，需要切换 |
| 并行 | 老板雇佣数据录入员分担工作 | 多任务真正同时运行（需要多核） |
| 竞态条件 | 银行转账和取款同时读取余额 | 未同步的并发操作导致数据不一致 |
| 互斥锁(Mutex) | 交易时锁定账户 | 【警告】解决竞态条件，但会降低可用性 |
| 信号量 | 浴室钥匙管理（二值/计数） | 控制资源访问数量的机制 |
| 死锁 | A等B完成，B等A完成 | 互相等待对方释放资源，导致永久阻塞 |

*** 5. 计算机安全

| 攻击类型 | 类比解释 | 防御思路 |
|----------|----------|----------|
| 暴力破解 | 尝试所有可能的钥匙 | 使用强密码、限制尝试次数 |
| 社会工程学 | 友好邻居骗取钥匙 | 提高安全意识，验证身份 |
| 安全漏洞 | 检查所有入口找弱点 | 定期更新、安全审计 |
| 木马 | 假装修水管的窃贼 | 不运行可疑程序 |
| Rootkit | 锁匠偷偷复制钥匙 | 难以检测，需要专业工具 |
| DDoS | 100人同时挤进书店 | 流量清洗、CDN、限流 |

密码学：
- 对称加密：双方持有相同钥匙
- 非对称加密：【重点】公钥加密，私钥解密。每人只需保管一个私钥，分发公钥即可

*** 6. 软件开发方法论

- 瀑布模型：像瀑布一样单向流动，阶段完成后无法返回
- 敏捷开发：持续迭代、协作、适应变化（Scrum、XP、Kanban）

*** 初学者常见问题速查

| 问题 | 一句话回答 |
|------|-----------|
| CS vs 编程 | 物理学家 vs 工程师 |
| 什么是编程 | 给听话但"愚蠢"的机器写精确指令 |
| 为什么不要打断开发者 | 进入心流像入睡，打断后需要重新进入 |
| Java vs JavaScript | 毫无关系（像车和地毯） |
| JS vs jQuery | jQuery是基于JS的库（ nerds vs 魔法师） |
| 框架 vs 库 | 你调用库，框架调用你 |
| 每天写多少行代码 | 无法衡量，甚至可能为负（还债） |
| 面向对象编程 | 对象是名词，方法是动词。对象封装复杂度，提供高层抽象接口 |
| 什么是API | 餐厅菜单：点菜→上菜，无需了解后厨 |
| SQL vs NoSQL | SQL像分 aisle 购物；NoSQL像按食谱一页存放所有信息 |
